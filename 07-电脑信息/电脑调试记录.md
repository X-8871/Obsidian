排查结果（根因已确认）：

**所有"自动重启"实际上是 NVIDIA RTX 4060 显卡驱动 TDR 超时导致的蓝屏（BugCheck `0x00000116` = `VIDEO_TDR_TIMEOUT_DETECTED`），系统因 `AutoReboot=1` 设置在蓝屏后自动重启，用户看到的现象就是"电脑突然重启"。**

证据：

1. 近 2 天共 15 次 `Kernel-Power 41`（意外关机），其中 12 次 BugCheck 为 `0x00000116`（GPU TDR），3 次为 `0x00000000`（纯断电）。
2. NVIDIA 驱动 `nvlddmkm` 在蓝屏前明确报告事件 ID 153：`BusReset TDR occurred on GPUID:100` 和 `Resetting TDR occurred on GPUID:100`。
3. 5 个 Minidump 文件全部确认 BugCheck `0x116`，参数 3 均为 `0xc000009a`（`STATUS_INSUFFICIENT_RESOURCES`）。
4. `0x116` 蓝屏在更换主板之前**从未出现过**。换主板前的崩溃全部是 `0x00000000`（断电）或其他无关错误码。

排除项：

- **外接显示器/HDMI 不是原因**：`0x116` 是 GPU 内部超时，不是显示信号问题。HDMI 问题只会导致无信号/黑屏，不会导致蓝屏。
- **Windows Update 不是原因**：近期仅有 Defender 病毒库和 OMEN 应用更新，无系统级驱动更新。
- **回退 NVIDIA 驱动无效**：已回退到 576.49，问题依旧，说明不是驱动软件版本的问题。
- **无 WHEA 硬件错误**：系统日志中无 WHEA-Logger 事件。

可能根因（按可能性排序）：

1. **新主板上焊接的 GPU 有硬件缺陷**（最可能）：HP Victus 16 的 RTX 4060 焊在主板上，换主板 = 换 GPU。问题完全在换主板后出现，高度指向新 GPU 有问题。
2. **新主板 PCIe 通道/供电不稳**：新主板供电模块可能无法稳定给 GPU 提供所需功率。参数 3 `0xc000009a`（`STATUS_INSUFFICIENT_RESOURCES`）也暗示资源不足。
3. **BIOS 兼容性**：BIOS F.30 可能与 GPU 驱动有时序问题。
