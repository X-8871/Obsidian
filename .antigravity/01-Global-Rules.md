> 这是 Antigravity 在该 Vault 中的最高优先级准则。无论执行任何任务，都必须遵守此文件中的习惯。

## 2. Markdown 格式偏好
**采用 [卡片盒双链 + 标签数据化 + 视觉排版] 混合流派**：
1. **视觉排版优先**：重度使用 Obsidian Callout 提示块 (如 `> [!tip]`, `> [!warning]`) 划分内容区块，并辅以 `==高亮文本==` 和 **粗体** 来突出核心重点，保持版面极高可读性。
2. **万物皆可双链**：重度依赖 `[[双向链接]]` 连接碎片化知识点（Zettelkasten思想），让知识网自由生长，减少对死板层级文件夹的依赖。
3. **标签与元数据驱动**：充分利用规范的 `#层级/标签`，并在需要聚合数据的页面内使用 Dataview 格式的内联字段 (例如 `Status:: Active`)，方便后期数据库式检索。

## 3. 文件命名与分类习惯
1. **强制中文命名**：所有笔记文件的物理名称都必须是中文的，以保证阅读直观性（如有必须使用英文的专有名词，可中英结合）。
2. **章节顺序命名**：无论是专业课还是技术学习，章节必须按照顺序命名（例如：带有序数前缀，如 `Module_1_...`, `Chapter_2_...`）。
3. **归类与建表优先**：在向知识库添加新文件时，必须先全局扫描现有目录树，优先放入已有对应的文件夹中；只有确认没有任何适配文件夹时，才去创建新文件夹。新文件或新文件夹产生后，必须立即将其关联到相应的双链网络，并更新对应的索引文件。
4. **Markdown格式归一化**：导入知识库的文件无论是纯文本 (txt)、Office 文档 (doc/docx)，还是各类代码脚本文件 (.py, .js 等)，**都必须先转换成 `.md` 格式文件**再放入 Vault 中（代码文件需以 Markdown 代码块的形式包裹存储），以保证所有知识点的统一管理和可读性。
