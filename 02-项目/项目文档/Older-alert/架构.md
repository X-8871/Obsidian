## 三、核心数据流：一次主循环的完整链路

```
main/app_main() 主循环 (每 2 秒一轮)
│
├─ 1. SensorHub_Read(&sensor_data)
│      └─ 逐一读取 AHT20/BH1750/MQ2/LD2410B
│      └─ 每传感器标记 _ok 有效标志
│      └─ 输出统一结构 sensor_hub_data_t
│
├─ 2. AppController_UpdateContext(&sensor_data)
│      └─ 更新休息上下文（低光+静止 → REST）
│      └─ 更新距离变化窗口
│      └─ 更新最近活动时间戳
│
├─ 3. 构建 risk_context_t（now_ms/inactive_ms/manual_sos/remind_timeout/rest）
│
├─ 4. RiskEngine_Evaluate(&sensor_data, &risk_context, &risk_result)
│      └─ MQ2 滑动平均滤波（5帧窗口）
│      └─ MQ2 轻度异常持续确认（DEMO: >=1000, 3秒）
│      └─ MQ2 高危险持续确认（DEMO: >=1300, 5秒）
│      └─ MQ2+温升联动判定
│      └─ 高温判定（DEMO: >=30°C, 30秒）
│      └─ 无活动判定（LD2410B: 有人存在+无明显运动）
│      └─ 休息上下文抑制无活动提醒
│      └─ 输出 risk_result_t（level + 各维度标志位）
│
├─ 5. AppController_Process(&sensor_data, &risk_result)
│      └─ 状态机转移：NORMAL ↔ REMIND ↔ ALARM ↔ SOS
│      └─ 状态变化时产出 event_key（供语音播报）
│
├─ 6. EventLog_Update(state, &sensor_data, &risk_result)
│      └─ 异常状态变化时记录现场快照
│
├─ 7. HttpAlertReporter_Process(state, &sensor_data, &risk_result)
│      └─ 状态变化 → 立即上报（EVENT）
│      └─ 每 10s → 遥测上报（TELEMETRY）
│      └─ POST → spectator0618.online:3000/api/alert
│
├─ 7.x RainMakerReporter_Process(...) — 备选上报链路
│
├─ 8. DisplayController_Update(state, &sensor_data, &risk_result)
│      └─ 更新 LVGL TFT 界面（NORMAL/REMIND/ALARM/SOS/OFFLINE）
│      └─ 显示时间、Wi-Fi 图标、传感器状态、按钮区
│
├─ 9. 内层循环 for(i=0; i<20; i++) — 每 100ms
│      ├─ 检测确认键长按 8s → 重置 Wi-Fi provisioning
│      ├─ AppController_Service()
│      │    ├─ 处理确认键/SOS 键事件
│      │    ├─ REMIND 超时升级为 ALARM
│      │    ├─ 冷却计时器管理
│      │    └─ 刷新 AlertController 声光输出
│      ├─ 消费待播报状态语音 event_key
│      ├─ DisplayController_Service(100) → 推进 LVGL tick
│      ├─ 语音上传测试：检测录音键 → SpeechUploader_RecordWavAndUpload()
│      └─ vTaskDelay(100ms)
│
└─ 10. 回到第 1 步，下一轮采样
```

## 四、状态机设计

```
                         ┌─────────────┐
        ┌────SOS键──────→│     SOS     │←── 最高优先级
        │                │  (深红/高频  │    锁存行为
        │                │   蜂鸣+LED) │
        │                └──────┬──────┘
        │                       │ 确认键解除
        │                       ↓
        │  ┌─────────┐    ┌─────────┐    长时间无活动/高温/MQ2轻度
        │  │ NORMAL  │───→│ REMIND  │←────────────────────────
        │  │ (绿色)  │←───│ (橙色)  │
        │  └────┬────┘    └────┬────┘
        │       │              │
        │       │    确认键    │ 超时 15s (DEMO) / 30min (REAL) 未确认
        │       │    解除      │
        │       │              ↓
        │       │         ┌─────────┐    MQ2高危险/MQ2+温升联动
        │       └────────→│  ALARM  │←──────────────────
        │                 │ (红色)  │
        │                 └────┬────┘
        │                      │
        │             确认键    │
        │             解除      │
        └──────────────────────┘
```

**状态优先级：** SOS > ALARM > REMIND > NORMAL

**关键状态转移规则：**

- NORMAL → REMIND：无活动超时 / 高温持续 / MQ2轻度异常持续
- REMIND → ALARM：超时未确认 / MQ2升级
- → SOS：按键直接触发（锁存，需确认键解除）
- ALARM → NORMAL：确认键（MQ2高危险/联动ALARM需额外等待回落稳定）
- 休息上下文（REST）：不作为独立显示状态，仅抑制无活动REMIND

## 七、关键设计决策

|决策|说明|
|---|---|
|**本地优先**|断网时 SOS/声光提醒/EventLog/确认解除仍可工作，云端仅作远程留痕|
|**DEMO/REAL 双参数**|编译期宏切换：DEMO (演示) 用短阈值速看效果，REAL 用真实居家参数|
|**MQ2 滑动平均 + 持续确认**|5帧窗口滤波，单次 ADC 超阈值不触发，必须持续 N 秒才报警|
|**半双工 I2S**|INMP441 与 MAX98357A 共用 BCLK/WS，录音→上传→释放→下载→播放顺序切换|
|**休息上下文抑制**|低光+静止进入 REST，抑制无活动 REMIND，避免夜间误报|
|**状态播报走云端 TTS**|本地仅保留 SOS 硬警报；其他状态过渡语音由云端 MiMo TTS 动态生成|
|**ST7735 兼容 ST7789 路径**|使用 `esp_lcd` 通用 st7789 面板驱动 + 显式 ST7735 初始化序列 + gap/mirror/swap_xy 补偿|
|**分区空间紧张**|`factory` 2MB，当前 `hello_world.bin` ≈ 1.9MB，剩余约 9-10%|
