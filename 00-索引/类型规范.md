# 类型规范

## 核心字段

重要笔记建议保留这三个字段：

```yaml
---
type: concept
domain: embedded
status: active
---
```

## type

- `project`：项目主文档。
- `project-plan`：项目实现计划、MVP、里程碑。
- `project-management`：分工、排期、协作。
- `project-research`：赛题、竞品、参考资料。
- `project-knowledge`：项目沉淀出的技术知识。
- `concept`：可复用概念。
- `learning-map`：学习路线。
- `learning-system`：能力提升体系。
- `workflow`：AI 或开发工作流。
- `guide`：使用指南、说明文档。
- `standard`：开发规范、团队规范。
- `plan`：个人规划。
- `habit-system`：习惯系统。
- `moc`：索引页。

## domain

- `embedded`：嵌入式总体。
- `embedded-networking`：网络、WiFi、以太网。
- `embedded-architecture`：架构、状态机、模块设计。
- `ai-workflow`：AI 工具链和 Codex 工作流。
- `personal`：个人规划。
- `fitness`：健身。
- `course`：课程。
- `vault`：Obsidian 库本身。

## status

- `active`：当前仍在使用。
- `draft`：草稿。
- `archived`：已归档。

## 使用原则

只给重要笔记加元数据。临时资料、摘录、Daily 流水账不需要强制补齐。
