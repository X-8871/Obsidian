# 阶段 1：卷积神经网络 (CNN) 的诞生

在我们在新手村搭建 `MySimpleNet` 时，我们使用的主要工具是 `nn.Linear` (线性层)。
但在《60-Minute Blitz》的实战中，我们要处理的是真实的彩色图片（比如猫和狗）。如果继续用 `Linear`，我们会遇到一个极其严重的问题。

---

## 1. 为什么 `Linear` 无法处理真实图片？

回想一下，在使用 `Linear` 之前，我们被迫做了一个极其残忍的动作：**`nn.Flatten()` (压面机)**。
如果一张图片上画了一只猫，猫的左眼在右眼的旁边，猫的鼻子在眼睛的下方。这种**“空间相邻关系”**是认出猫的最核心线索。
但是，`Flatten` 会把一张二维的方形图片，生硬地拉成一根极长极长的一维面条！
在面条里，猫左眼和鼻子的像素可能被隔开了十万八千里。大厨看着这根面条，根本无法理解“眼睛下面是鼻子”这种空间关系。

**这就是我们需要抛弃（至少是在最初层抛弃）`Linear` 的原因！**

---

## 2. 救世主登场：`nn.Conv2d` (二维卷积层)

为了让机器拥有“真正的视觉”，科学家发明了 **卷积神经网络 (Convolutional Neural Networks, CNN)**。它的核心大厨名叫 `nn.Conv2d`。

`Conv2d` 绝对不会把图片压扁！它会拿着一个类似于“放大镜”的微型窗口（通常是 3x3 或 5x5 的像素格，在术语里叫 **Kernel / 卷积核**），在原始的二维图片上**从左到右、从上到下地滑动扫描**。

### 扫描在找什么？
*   有的放大镜（卷积核）负责找**横向的边缘**。
*   有的负责找**竖向的边缘**。
*   有的负责找**特定的颜色斑块**。

当这块 3x3 的放大镜扫过带有猫耳尖角的区域时，如果刚好和放大镜要找的形状对上了，它就会疯狂打高分！因为它是以 2D 的方式在图片上滑动的，所以它**完美地保留了所有像素的空间相邻关系**！

### 参数解析
```python
import torch.nn as nn

# 召唤一个卷积大厨
conv1 = nn.Conv2d(in_channels=3, out_channels=6, kernel_size=5)
```
*   **`in_channels=3` (输入通道)**：真实世界的彩色图片都有 R、G、B 三个颜色通道，所以这里必须填 3。
*   **`out_channels=6` (输出通道)**：你想让这位大厨拿着多少把不同的“放大镜”去扫描？写 6 就代表它会提取出 6 种不同的特征画面（比如边缘画面、色彩画面等）。
*   **`kernel_size=5` (感受野)**：放大镜的尺寸是 5x5 的像素格。

---

## 3. 终极瘦身法宝：Max Pooling (最大池化)

在用 `Conv2d` 扫完图片后，图片里充满了各种特征。但如果图片是 1000x1000 这么大，计算量会直接让显卡原地爆炸。

这时候，我们需要一个瘦身法宝：**`nn.MaxPool2d` (最大池化层)**。
你可以把它理解为**“精华提取器 / 马赛克机器”**。

```python
import torch.nn.functional as F

# 经常在 forward 函数里这样用
x = F.max_pool2d(x, (2, 2))
```
**它干了什么？**
它在图片上每 2x2 的四个小格子里，**只保留得分最高的那一个像素，扔掉其他三个**！
*   **好处 1**：图片的宽和高瞬间缩小了一半！数据量直接锐减为原来的四分之一，极大减轻了显卡负担。
*   **好处 2 (抗干扰)**：只要那四个格子里有一个猫耳朵的特征，它就能被保留下来。这意味着即使猫在图片里稍微往左或往右挪了几个像素，大厨依然能敏锐地认出它！这在术语里叫“平移不变性”。

---

## 4. CNN 经典工作流水线

全世界上到 ChatGPT，下到人脸识别闸机，经典的 CNN 都是按这个死套路组装的：

1. **提取特征 (看图)**：连续使用好几对 `[Conv2d -> ReLU -> MaxPool2d]`。就像是用越来越多的放大镜一遍遍地看，同时把图片越缩越小、提炼精华。
2. **压扁面条 (Flatten)**：等图片被缩小到只有几乘几的马赛克大小时（此时里面全是最顶级的猫狗特征），我们在最后时刻把它压扁成面条。
3. **投票表决 (Linear)**：用你最熟悉的苦力小厨师 `nn.Linear` 接收面条，对里面的特征进行打分，给出最终是哪一类的预测结果。
