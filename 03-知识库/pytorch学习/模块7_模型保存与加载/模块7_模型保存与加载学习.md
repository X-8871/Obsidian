# 模块 7：保存与加载模型 (Save and Load the Model)

在前面几章中，我们辛辛苦苦招募大厨，又请品菜师和教练在后厨足足特训了 3 天，大厨终于把准确率提升到了及格线。
但这往往只是开始。在真实的企业级 AI 训练中，像 ChatGPT 这样的大厨，要在带有上万张顶级显卡的超级后厨里被连续狂训好几个月，消耗上千万美元的电费！

如果训练结束那一刻，程序员不小心把电脑关了……大厨脑子里的记忆（运行内存中的数据）会瞬间清零，老板会直接把你祭天。

所以，我们需要学会极其重要的一步：**给大厨拍脑部 X 光片，把记忆以文件的形式死死地刻录进硬盘里！** 明天开机，只要读盘，就能立刻“复活”一位满级大厨。

PyTorch 提供了两套不同的备份方案。

---

## 方案一：只保存“脑神经数值”（官方强烈推荐！）

这也是全世界 99% 的高级炼丹师都在用的正统方案。

### 1. 记忆的本体：`state_dict()`
大厨（模型）的肚子里有一本极其神秘的字典，叫作 `state_dict`（状态字典）。
这本字典里没有任何做菜的动作（代码逻辑），它纯纯粹粹只记录了一样东西：**所有大大小小的神经元，在被教练狂殴之后，最终定格下来的那个数值（权重 w 和偏置 b）。**

我们可以用一句话，把这本纯粹的数字字典剥离出来，存成一个扩展名为 `.pth`（PyTorch 的缩写）的文件：

```python
import torch
import torchvision.models as models

# 假设这个 model 就是我们刚才辛辛苦苦训好的满级大厨
# 把他的脑子（状态字典）抽出来，存进硬盘里，取名 model_weights.pth
torch.save(model.state_dict(), 'model_weights.pth')
print("大厨的记忆已安全存入硬盘！")
```

### 2. 复活大厨（加载参数）
第二天，电脑重新开机。内存是空的，所以你第一步必须得**重新拿图纸造一个空壳大厨**。
然后，把你昨天存进硬盘的记忆文件读出来，**强行灌入（Load）空壳大厨的脑子里**：

```python
# 1. 拿图纸重新造一个失忆的、只会瞎蒙的空壳大厨
model = NeuralNetwork() 

# 2. 从硬盘里读出昨天的记忆字典，并灌装给大厨
model.load_state_dict(torch.load('model_weights.pth', weights_only=True))
print("满级记忆灌装完毕！大厨复活！")
```

### 3. 营业前的一句极其重要的咒语：`model.eval()` ⚠️
当大厨复活后准备去大堂干活（推理/测试阶段）之前，你必须对着他念一句极其重要的咒语：

```python
model.eval() # 开启营业模式 (Evaluation Mode)
```

**为什么一定要写这句？**
在深度学习里，有些神经层（比如我们在进阶课会学到的 Dropout 层或者 BatchNorm 层），它们在“后厨被特训时”的行为，和“在大堂正式营业时”的行为是完全不一样的！
*   如果不写这句，大厨会误以为自己还在被特训，做菜的动作会极度诡异变形。
*   写了这句，就等于通知大厨：**“别练了，现在是在给客人出餐，拿出你最稳的最终形态！”**
*(不要把它和我们上一章学的 `torch.no_grad()` 混淆。`no_grad` 是关掉底层的自动算梯度探头，而 `model.eval()` 是切换大厨本人的工作模式。这两句话在正式营业时往往是双剑合璧、一起使用的。)*

---

## 方案二：把大厨整个人塞进硬盘里（官方不推荐）

有时候程序员很懒。不仅想存脑子（数值），还想把整个大厨的肉体结构（类的定义图纸、所有的逻辑层）一并打包塞进硬盘里。

```python
# 极其粗暴，不用 state_dict，连肉体带灵魂一起存
torch.save(model, 'model.pth')
```

加载的时候更粗暴，连空壳都不用造，直接从硬盘里解压出一个完整的大厨：
```python
# 直接复活完整大厨
model = torch.load('model.pth', weights_only=False)
```

**为什么这套方案极其危险，被官方嫌弃？**
因为这种打包方式（用的是 Python 底层的 pickle 序列化），把**代码所在的文件路径和类的名字**死死地绑定进去了！
如果你今天把保存的模型发给了你的朋友，或者你把代码稍微挪了一个文件夹，甚至给类改了个名字。当你再次尝试 `torch.load` 时，系统会因为找不到原始代码的位置而当场爆炸崩溃。

所以，在实战中，我们永远只采用**方案一（只保存并加载 `state_dict` 参数）**，这样你的数据和你的逻辑代码永远是清清爽爽分离的。

## 总结：打通最后的闭环
从现在起，你的深度学习宏大流程终于彻底画上了完美的句号：
1. **数据准备** (Dataset & DataLoader)
2. **搭建网络** (nn.Module)
3. **训练大厨** (Train Loop: Forward -> Loss -> Zero Grad -> Backward -> Step)
4. **考核大厨** (Test Loop)
5. **保存果实** (`torch.save(model.state_dict())`)
6. **随时复活** (`model.load_state_dict()`) 并开启营业 (`model.eval()`)

## 4. Q&A 补充精华（基于你的深度提问）

在互动中，我们填平了实战写代码时极其容易踩的坑：

### 4.1 `model_weights.pth` 到底存到哪里去了？
如果不写盘符，这叫**相对路径**。文件会保存在你的**“当前工作目录” (CWD)**，通常就是你跑这段 Python 代码所在的那个文件夹。如果你想存到桌面，可以写**绝对路径**。

### 4.2 Windows 路径的惊天大雷（`/` 和 `\`）
在 Windows 系统的路径里用的是反斜杠 `\`。但在 Python 字符串里，`\` 是用来变魔术的转义符（比如 `\n` 是回车，`\t` 是 Tab）。
如果你写 `'C:\new_folder\test.pth'`，Python 会把它理解成一堆换行符而导致当场爆炸！
**破解绝招**：
1. 封印魔法：在最前面加 `r`（比如 `r'C:\new_folder\test.pth'`）。
2. **极其推荐的跨平台写法**：无视 Windows，直接把地址里的斜杠全换成正斜杠 `/`（比如 `'C:/new_folder/test.pth'`），Python 底层会自动帮你翻译！

### 4.3 既然没写过模式，为什么能调用 `model.eval()`？
因为你在类定义的开头写了 `super().__init__()`，从而继承了干爹 `nn.Module` 的亿万家产。干爹内部自带了几百个隐藏开关，其中就包括 `self.training`（训练模式 / 营业模式）。

### 4.4 为什么连 `Linear` 都不受影响，我们却还要死记硬背加上 `model.eval()`？
虽然我们今天写的 `Linear` 和 `ReLU` 在白天晚上干活是一样的。但在进阶网络中，有一些行为“极其变态”的层：
*   **比如 `nn.Dropout`（随机失活层）**：在训练模式下，它会为了抗压，随机“打晕”后厨里 50% 的脑细胞；如果不写 `eval()` 关掉它，大厨在大堂营业时也会随机脑死亡 50%，导致菜品灾难！
### 4.5 为什么要强制写 `weights_only=True`？（安全排雷指南）
如果在读盘 `torch.load()` 时不加这句话，你可能会引爆计算机安全领域的**天坑漏洞**！
PyTorch 底层存盘用的是 Python 自带的 `pickle` 打包技术，它可以打包纯数字，也可以打包一段**可执行的危险代码**。
如果你从网上随便下载了一个不怀好意的黑客制作的 `.pth` 模型，你不写 `weights_only=True` 就直接 `load`，它隐藏的木马代码就会在你的电脑里执行（比如删除全盘文件）！
**这句咒语就相当于门口的“安检扫描仪”**，加上它意味着：“我只允许纯粹的神经元数字通过，任何复杂的代码结构一律拦截报错！”这是你在工业界保命的铁律！
