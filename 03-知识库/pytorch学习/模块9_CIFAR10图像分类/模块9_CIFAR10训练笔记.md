## 1. 我们的食材：CIFAR-10 数据集
在上一篇实战里，我们用的 `FashionMNIST` 是黑白图片（单通道，`1x28x28`）。
而今天我们要挑战的 CIFAR-10 是**真实的彩色照片**。
*   形状：`3 x 32 x 32`（3 个颜色通道：红、绿、蓝）。
*   种类：10 个类别（飞机、汽车、鸟、猫、鹿、狗、青蛙、马、船、卡车）。

因为是 3 通道输入，所以待会儿我们定义第一个卷积厨师时，**`in_channels` 必须写 3**！

---

## 2. 真实工业界的网络结构：双重压榨与层层递进
刚才你写了 1 个卷积大厨，但真实的 AI 往往要复杂得多。为了看懂复杂的图片，我们通常采用**“卷积-池化-卷积-池化-线性表决”**的经典组合。

```python
class Net(nn.Module):
    def __init__(self):
        super().__init__()
        # 1. 第一个卷积大厨 (进3出6)
        self.conv1 = nn.Conv2d(3, 6, 5)
        # 2. 瘦身机
        self.pool = nn.MaxPool2d(2, 2)
        # 3. 第二个卷积大厨 (进6出16，提取更高级的特征)
        self.conv2 = nn.Conv2d(6, 16, 5)
        
        # 4. 最后把这些高级特征压扁，交给 3 个线性苦力去投票
        # 因为经过两次 5x5卷积 和两次 2x2池化，32x32 的图片最后被缩成了 5x5 的方块
        # 16 个通道 * 5 * 5 = 400 根面条
        self.fc1 = nn.Linear(16 * 5 * 5, 120)
        self.fc2 = nn.Linear(120, 84)
        self.fc3 = nn.Linear(84, 10) # 最后必须输出 10 (因为有10种物体)
```
别被这一长串吓到，它的流水线逻辑极为顺畅：
**图片进锅 -> 提取浅层特征 -> 缩减尺寸 -> 提取深层特征 -> 再次缩减 -> 压扁成面条 -> 线性打分！**

## 3. 把大厨和菜端进终极微波炉：GPU 加速 (Device)
既然是彩色大图，大厨在 CPU (普通炉灶) 上炒菜的速度会极其缓慢。
我们要使用 PyTorch 的核心魔法：**设备转移 `.to(device)`**。

```python
# 1. 探测你的电脑有没有高级微波炉 (NVIDIA 显卡)
device = torch.device('cuda' if torch.cuda.is_available() else 'cpu')
print(f"当前使用的微波炉是: {device}")

# 2. 把大厨强行塞进微波炉
model.to(device)

# 3. 在 train_loop 里，把传菜员端上来的每一盘菜，也扔进微波炉！
for batch, (X, y) in enumerate(dataloader):
    X, y = X.to(device), y.to(device) # 极其重要！大厨在微波炉里，菜必须也送进去！
```
**死记硬背防坑指南**：模型在哪里，数据就必须在哪里！如果在 CPU 上的模型去吃 GPU 里的张量数据，系统会当场爆炸报错。
