# Python 常见标点符号避坑口诀 (CheatSheet)

对于初学 Python (特别是从 C/C++ 转来) 的新手，标点符号往往是最容易踩坑的地方。只要记住以下四个符号的口诀，就能看懂 99% 的基础报错！

---

## 1. 冒号 `:` —— “我要开辟新地盘了，下面跟着我缩进！”

*   **什么时候用**：只要你看到代码里出现了“宣誓主权”的控制词语（比如定义一个类 `class`，定义一个函数 `def`，写一个条件判断 `if/else`，或者写一个循环 `for/while`），这一行的结尾**必须**用冒号。
*   **口诀**：冒号就像是领导开会说：“今天说三件事（冒号）：” —— 然后下面必定跟着几行向右缩进（空 4 格）的详细指示。
*   **正确示例**：
    ```python
    def forward(self, x):  # 👈 看到 def，末尾必加冒号
        x = self.flatten(x)
        return x
    ```

---

## 2. 圆括号 `()` —— “我要执行动作了！” 或者 “把这些打包！”

在 Python 里，它主要干两件事：

*   **场景A：动作启动按钮（调用函数）**
    *   如果你光写 `item`，你只是用手指着“剥壳机”这台机器，机器不会动。
    *   如果你写 `item()`，相当于你**重重地按下了机器上的绿色启动按钮**，机器才真正开始干活。同理，`print()`、`model()` 统统都是按下了执行按钮。
*   **场景B：打包包裹 (元组)**
    *   你想把多个东西当成一个整体传递，比如我们在数据加载里学的 `(img, label) = dataset[0]`。
*   **常见报错**：如果不小心丢了括号，比如写了 `print(y_pred.item)`，屏幕上不会出数字，只会打印出一长串类似 `<built-in method...>` 的乱码，意思是“你指了指这台机器，但没按启动键”。

---

## 3. 逗号 `,` —— “我和他是并列的兄弟”

*   **什么时候用**：逗号永远用来**隔开并列的东西**。
*   **场景**：
    *   在函数括号里传递好几个参数：`nn.Linear(128, 10)`（这台机器既需要知道进口 128，又需要知道出口 10，两者是并列关系）。
    *   在一行代码里接手多个包裹：`images, labels = next(...)`
    *   在列表里放好几个数字：`[1, 2, 3]`

---

## 4. 分号 `;` —— “在 Python 里，请你当它不存在！”

*   **什么时候用**：在 C 语言或者 Java 里，每一行代码的结尾都必须加上分号当“句号”。**但在 Python 里，回车换行就是天然的句号！**
*   **口诀**：**忘掉分号吧！** 除非你想强行把两行代码挤在一行里写（比如 `a = 1; b = 2`），但这种写法在 Python 极其罕见且不推荐。
