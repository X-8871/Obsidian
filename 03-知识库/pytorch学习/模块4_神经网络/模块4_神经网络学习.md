# 模块 4：搭建神经网络 (Build the Neural Network)

神经网络就像是一个极其复杂的超级大厨，它由许多个负责切菜、炒菜、调味的小帮手（层 Layer）串联而成。
在 PyTorch 中，所有的神经网络模块和组件都存放在 `torch.nn` 这个魔法工具箱里（`nn` 就是 Neural Network 的缩写）。

本章，我们将像搭乐高积木一样，一层一层地拼装出一个能够识别 FashionMNIST 衣服分类的赛博大脑！

---

## 1. 万物皆可 nn.Module

在 PyTorch 中，所有的神经网络模型都**必须**认 `nn.Module` 作“干爹”（继承它）。
这和我们在第 2 章自定义 Dataset 的逻辑简直如出一辙！只要你继承了 `nn.Module`，你写的类就拥有了所有的超能力（比如自动管理参数、支持 GPU 加速等）。

一个合格的神经网络类，必须具备两个灵魂函数：
1. **`__init__` (初始化/备菜区)**：在这里，我们把所有需要用到的大锅、切菜机（各种神经层）提前买好，登记在册。
2. **`forward` (前向传播/流水线)**：在这里，我们规定这些锅碗瓢盆的先后顺序，明确数据进来之后，先经过哪台机器，再经过哪台机器。

```python
import torch
from torch import nn

# 获取运算设备（GPU 或 CPU）
device = (
    "cuda" if torch.cuda.is_available()
    else "mps" if torch.backends.mps.is_available()
    else "cpu"
)
print(f"当前正在使用 {device} 构思我们的网络")

# 定义我们的神经网络类！
class NeuralNetwork(nn.Module): # 认 nn.Module 作干爹
    def __init__(self):
        super().__init__() # 拜干爹的固定仪式
        
        # 1. 压面机：把二维图片压扁成一维条条
        self.flatten = nn.Flatten()
        
        # 2. 核心炒菜流水线 (Sequential 把几个层强行捆绑在一起)
        self.linear_relu_stack = nn.Sequential(
            nn.Linear(28*28, 512), # 第一个厨师：接手 784 根面条，揉成 512 根
            nn.ReLU(),             # 第一瓶神仙酱料 (激活函数)
            nn.Linear(512, 512),   # 第二个厨师：接手 512 根面条，重新揉成 512 根
            nn.ReLU(),             # 第二瓶神仙酱料 (激活函数)
            nn.Linear(512, 10),    # 第三个厨师：接手 512 根面条，精炼出 10 个最终打分
        )

    # 规定流水线顺序：数据 x 进来了该怎么走？
    def forward(self, x):
        x = self.flatten(x)                # 先过压面机
        logits = self.linear_relu_stack(x) # 再过那条核心炒菜流水线
        return logits                      # 吐出 10 个类的打分
```

## 2. 深入拆解大厨的秘密武器 (Model Layers)

我们来看看 `__init__` 里那些奇奇怪怪的零件到底在干什么。

### 武器 1：`nn.Flatten()` (展平层 / 压面机)
*   **用途**：大批量送进来的图片是 2D 的（28行 x 28列 的像素）。但是，神经网络的神经元是排成一条直线的！
*   **动作**：所以 `nn.Flatten()` 像压面机一样，把一个 28x28 的二维矩阵，强行压平成一个包含 `784` 个像素值的一维数组。

### 武器 2：`nn.Linear(in_features, out_features)` (线性层 / 负责干苦力的厨师)
*   **用途**：这是整个神经网络**最核心**的计算部分。
*   **动作**：它内部维护着极其庞大的“权重矩阵（Weights）”和“偏置（Bias）”。比如 `nn.Linear(784, 512)` 意味着它要把进来的 784 个像素值，通过疯狂的矩阵乘法和加法（小学数学的 `y = wx + b` 的究极放大版），转换成 512 个全新的数字。

### 武器 3：`nn.ReLU()` (激活函数 / 神仙酱料)
*   **用途**：线性层 (`nn.Linear`) 虽然算力强悍，但它只能做极其死板的“直线/平面操作”。如果没有 `ReLU`，你不管叠加多少层 `Linear`，数学上都可以证明它们能被合并成单单 1 层！
*   **动作**：`ReLU` 就像是一勺极其特别的酱料（**引入非线性**）。它的逻辑简单得令人发指：**“只要进来的数字是负数，一律归 0；只要是正数，原样放行！”**。就这么一点微小的拐弯/非线性，赋予了神经网络去拟合世界上极其复杂的不规则曲线的能力。

### 武器 4：`nn.Sequential()` (捆绑器 / 流水线传送带)
*   **用途**：如果你有很多厨师和很多酱料，一个个写调用太麻烦了。`nn.Sequential` 把他们像糖葫芦一样串在一起。数据只要扔进这个传送带的头，就会自动依次穿过所有的厨师和酱料，从尾巴出来。

## 3. 把大厨造出来并试营业

大厨的图纸画好了，现在我们把它实例化（造出来），并放到厨房（CPU 或 GPU）里：

```python
model = NeuralNetwork().to(device)
print(model) # 打印大厨的内部结构
```

接着，我们随便造假数据喂给它吃：
```python
# 凭空捏造一张全 1 的假图片 (模拟前面学过的 1 个 batch_size)
X = torch.rand(1, 28, 28, device=device) 

# 核心动作：直接把数据喂给大厨 (不需要手动调用 model.forward(X))
logits = model(X) 
print(f"大厨最初做出的 10 个打分: {logits}")

# 这些打分通常很大或者很小（甚至有负数），为了看出它到底觉得衣服是哪一类的概率最大，
# 我们再加一道最后工序：Softmax
pred_probab = nn.Softmax(dim=1)(logits)
y_pred = pred_probab.argmax(1)
print(f"模型预测的分类结果是第 {y_pred.item()} 种衣服！")
```
*(注意：因为我们刚刚造出大厨，根本还没开始教他（训练），所以他现在的初始权重是纯随机的，它的预测也是在盲目瞎猜。)*

## 4. 窥视大厨的脑细胞 (Model Parameters)

大厨（网络）的肚子里，到底装了多少参数呢？我们可以用 `model.named_parameters()` 把它的底裤给扒出来看一看。

```python
print(f"模型结构: {model}\n\n")

for name, param in model.named_parameters():
    print(f"层名字: {name} | 大小(Size): {param.size()} | 具体数值的片段: {param[:2]} \n")
```
你会看到诸如 `linear_relu_stack.0.weight` 这样的名字，它们的大小正好是 `[512, 784]`！这就是刚才提到的底层大矩阵！整个网络的成千上万个参数，就在这里静静地等待我们下一章去训练它们。

## 5. Q&A 补充精华（基于你的深度提问）

在我们的互动中，我们进一步拆解了神经网络内部各个组件的底层逻辑：

### 5.1 `Sequential` 传送带的本质
如果没有 `Sequential`，我们在 `forward` 里就需要手动把数据 `x` 在每个层之间递来递去（`x = layer1(x)` -> `x = layer2(x)`...）。`Sequential` 是一个能把多个层串联打包的流水线传送带。有了它，在 `forward` 里只要写一句 `logits = self.my_conveyor_belt(x)` 就能自动穿透所有层，极大地偷了懒并保持代码整洁。

### 5.2 为什么 `logits` 不在 `forward(self, x)` 的参数里？
*   `x` 是从外面扔进来的生肉（输入参数）。
*   `logits` 是生肉经过机器（神经网络）内部加工后，在函数体内现造出来的“香肠”（局部变量）。
在深度学习的黑话中，模型吐出的、还没转化为百分比概率的原始粗糙打分，习惯被命名为 **Logits**。最后 `return logits` 是把香肠交还给调用者。

### 5.3 为什么必须要有 `ReLU`？（极其核心）
*   **数学坍缩**：`Linear` 的本质是一次线性代数运算（`w*x+b`）。如果不加 `ReLU`，不管你串联多少个 `Linear` 层，数学上它们全都可以被化简（乘法分配律）成单独的一个大 `Linear` 层。这意味着 100 层的纯线性网络和 1 层的威力完全一样，只能画一条直线。
*   **折纸艺术**：`ReLU`（遇到负数就强行归 0）这个动作，在数学上叫作“引入非线性”。相当于在笔直的钢筋上强行打断出一个“关节”。成千上万个带关节的短钢筋拼接在一起，就能完美勾勒出世界上的任何极其复杂的弯曲轮廓（通用拟合定理）。

### 5.4 打印模型 (`print(model)`)
当模型被实例化（造出来）后，直接 `print(model)` 可以打印出大厨的“体内透视图”。你能清晰地看到每一层叫什么名字，接收多少根面条（`in_features`），吐出多少根面条（`out_features`），是抓虫（Debug）和检查网络有没有连错的最佳手段。

### 5.5 `Softmax` 与 `argmax` 的职责交接
*   **`nn.Softmax(dim=1)`**：它就像一个榨汁机，把原本乱七八糟的 `logits` 强行压缩成总和为 100% 的概率分布。此时它生成并返回了一个**全新的张量（Tensor）**。
*   **`.argmax(1)`**：**它根本不是 `Softmax` 的函数！** 它是 PyTorch 张量家族每个人生来就自带的绝技。我们是对刚才 `Softmax` 生成的那杯果汁（Tensor）使用了 `.argmax(1)` 技能，让它告诉我们：这排概率里，最大的那个数值坐在第几号座位（Index）上。这也就是模型最终拍板的分类结果。
