## 一、 基于计算图的完整推导 (前向与反向传播)

![[ScreenShot_2026-05-26_160731_614.png]]
### 1. 前向传播 (Forward Propagation：从 $A^{[0]}$ 到 $\hat{Y}$)

数据从输入端进入，雷打不动地经历“线性组合 $\rightarrow$ 非线性激活”的过程。

- **第一层（隐藏层）：**
    
    - 线性得分：$Z^{[1]} = W^{[1]}A^{[0]} + b^{[1]}$
        
    - 激活输出：$A^{[1]} = g^{[1]}(Z^{[1]})$
        
- **第二层（输出层）：**
    
    - 线性得分：$Z^{[2]} = W^{[2]}A^{[1]} + b^{[2]}$
        
    - 预测概率：$A^{[2]} = g^{[2]}(Z^{[2]}) = \sigma(Z^{[2]})$
        

### 2. 反向传播 (Backward Propagation：推导 6 个核心梯度公式)

我们从最右侧的损失函数 $\mathcal{L}$ 开始，沿着红线用链式法则向左推导。

**阶段一：输出层（第 2 层）追责**

1. **求 $dZ^{[2]}$（误差源头）**：损失函数对 $Z^{[2]}$ 求导。它是经过对数损失和 Sigmoid 导数抵消化简后得到的最简形态：
    
    $$dZ^{[2]} = A^{[2]} - Y$$
    
2. **求 $dW^{[2]}$**：$Z^{[2]}$ 对 $W^{[2]}$ 求导，剩下送入本层的 $A^{[1]}$。为了维度对齐和全样本平均：
    
    $$dW^{[2]} = \frac{1}{m} dZ^{[2]} A^{[1]T}$$
    
3. **求 $db^{[2]}$**：
    
    $$db^{[2]} = \frac{1}{m} \text{np.sum}(dZ^{[2]}, \text{axis}=1, \text{keepdims=True})$$
    

**阶段二：隐藏层（第 1 层）追责**

4. **求 $dZ^{[1]}$（最难跨越的一步）**：第 2 层的误差顺着连线（$W^{[2]}$）退回来，再穿过第 1 层的激活函数大门（乘上导数）。

$$dZ^{[1]} = W^{[2]T} dZ^{[2]} * g^{[1]'}(Z^{[1]})$$

_(注：这里 `*` 代表逐元素相乘，因为激活函数的导数只在神经元内部局部生效。)_

5. **求 $dW^{[1]}$**：拿着算好的本层误差，乘上上一层送来的产品（即输入 $A^{[0]}$）：

$$dW^{[1]} = \frac{1}{m} dZ^{[1]} A^{[0]T}$$

6. **求 $db^{[1]}$**：

$$db^{[1]} = \frac{1}{m} \text{np.sum}(dZ^{[1]}, \text{axis}=1, \text{keepdims=True})$$

## 二、 激活函数 (Activation Functions) 与其导数

**什么是激活函数？**

如果不用激活函数（或者使用线性激活 $g(z)=z$），无论神经网络叠多少层，它本质上都在做一堆线性运算的组合，最终效果只等同于一个单层逻辑回归，**无法处理任何复杂的非线性问题**。激活函数的作用，就是给网络注入“非线性变形”的能力。

由于在反向传播计算 $dZ$ 时，我们必须用到激活函数的导数 $g'(z)$，所以掌握它们的求导极其关键。

常见的四种非线性激活函数及导数：

### 1. Sigmoid

- **公式**：$a = \frac{1}{1 + e^{-z}}$
    
- **导数 $g'(z)$**：$a(1-a)$
    
- **适用场景**：目前**绝对不建议用在隐藏层**（会导致梯度消失）。仅在二分类问题的**输出层**使用，用来输出 0 到 1 之间的概率。
    

### 2. Tanh (双曲正切)

- **公式**：$a = \frac{e^z - e^{-z}}{e^z + e^{-z}}$
    
- **导数 $g'(z)$**：$1 - a^2$
    
- **适用场景**：本质上是向下平移后的 Sigmoid，取值范围是 $[-1, 1]$。它的效果几乎总是比 Sigmoid 好，因为它的输出是**以 0 为中心 (Zero-centered)** 的，这能让下一层学习得更快。但它同样存在两端梯度消失的问题。
    

### 3. ReLU (线性整流函数)

- **公式**：$a = \max(0, z)$
    
- **导数 $g'(z)$**：当 $z < 0$ 时为 $0$；当 $z \ge 0$ 时为 $1$。
    
- **适用场景**：**隐藏层的默认首选**。计算极快，且在正半轴导数恒为 1，彻底拯救了深层网络的梯度消失问题。
    

### 4. Leaky ReLU

- **公式**：$a = \max(0.01z, z)$
    
- **导数 $g'(z)$**：当 $z < 0$ 时为 $0.01$；当 $z \ge 0$ 时为 $1$。
    
- **适用场景**：为了解决 ReLU 在负半轴导数为 0 导致的“神经元死亡（永远不更新）”问题而提出。虽然理论上更好，但在实际工程中，普通的 ReLU 依然占据统治地位。
    

## 三、 为什么必须随机初始化 (Random Initialization)？

在逻辑回归中，我们可以把权重 $W$ 全部初始化为 0。但在多层神经网络中，**绝对不能把 $W$ 初始化为全 0（或者全是同一个常数）**。

### 1. 致命后果：对称性崩溃 (Symmetry Breaking Problem)

假设你把隐藏层所有的 $W^{[1]}$ 都初始化为 0。

1. **前向传播**：第一层的两个神经元算出来的 $Z$ 和 $A$ 将会一模一样（因为权重一样）。
    
2. **反向传播**：因为输出一样，它们收到的误差回传 $dZ^{[1]}$ 也会一模一样。
    
3. **参数更新**：它们算出的梯度 $dW^{[1]}$ 相同，更新后的 $W$ 依然一模一样。
    

这意味着，无论你训练多久，隐藏层的多个神经元都在做完全相同的工作，计算完全相同的特征。这叫做“完全对称”，此时拥有 100 个神经元的隐藏层，其实和只有 1 个神经元没有任何区别。

### 2. 正确做法：小随机数初始化

为了打破这种对称性，我们需要给 $W$ 赋予非常小的随机值（偏置 $b$ 初始化为 0 没关系）。

- **代码实现**：`W = np.random.randn(n_out, n_in) * 0.01`
    
- **为什么要乘 0.01 (为什么必须是小数字)？**
    
    如果权重初始化得很大，算出来的初始 $Z$ 就会很大或者很小。如果网络碰巧使用了 Sigmoid 或 Tanh，大的 $Z$ 会直接落入激活函数两端异常平缓的区域（曲线变平），此时梯度 $g'(z)$ 趋近于 0，**网络在第一步训练时就会直接卡死，学不动了**。因此，初始化必须既随机，又要尽量小。
