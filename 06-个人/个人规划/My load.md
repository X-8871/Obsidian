时代趋势：
单点技能会贬值，系统型人才会升值。  
只会生成代码的人会变廉价，能用 AI 做出现实系统的人会变强。

三个就业方向
1. **IoT 应用开发工程师**  
    偏设备联网、数据上传、App/网页/云平台对接。
    
2. **智能硬件系统集成工程师**  
    偏传感器、主控、通信、电源、外壳、调试，把东西做出来。
    
3. **AI+硬件产品原型负责人**  
    偏需求洞察、功能设计、快速原型、比赛/项目/商业化表达。


岗位关键词是：IoT应用开发、嵌入式应用开发、智能硬件原型开发、物联网设备开发、AI+硬件项目开发。

- 南京/苏州/无锡：物联网、工业设备、仪器仪表、智能硬件、中小型研发公司
- 杭州：IoT 平台、AI 应用、智能硬件，但节奏可能更强
- 合肥：半导体、新能源、智能设备，但岗位更偏硬件/制造/测试的也多

**就业优先，考研备选；AI 深度拥抱，但从代写工具改成学习教练；主赛道押 AI + IoT 智能硬件原型；底层嵌入式只补到能读懂、能改、能调试的程度；用老人看护器作为第一块真正属于你的作品。**

**第一阶段：现在到大二结束，拿回代码掌控感**  
每天 2 小时：

- 1 小时：C 语言 + ESP-IDF 小题
- 40 分钟：读老人看护器现有代码
- 20 分钟：用 AI 复盘，让它考你

你可以先抄代码，这没问题。但要用“三遍法”：

1. 第一遍照抄，跑通。
2. 第二遍遮住，自己写。
3. 第三遍改需求，比如把按钮报警改成定时报警、串口输出改成 JSON。

**第二阶段：暑假，做出一个完整 IoT 闭环**  
目标不是炫技，而是完成：

ESP32 采集数据 → 判断状态 → WiFi 联网 → HTTP/MQTT 上传 → 手机/网页查看 → 报警。

你只要能独立讲清这条链路，就已经比“只会让 AI 写代码”的人强一截。

**第三阶段：大三上，做简历项目和实习准备**  
老人看护器不要只当比赛项目，要打磨成你的代表作：

- 系统架构图
- 功能演示视频
- GitHub/Gitee 代码仓库
- 项目文档
- 遇到的问题和解决方案
- AI 在项目中如何辅助，而不是代替你


可以。我们把路线拆成 **4 个阶段**。每一阶段都必须有“过关标准”，不是感觉自己学了，而是能拿出东西证明。

**阶段一：现在到大二结束，拿回代码掌控感**

目标：从“AI 写什么我用什么”，变成“AI 写的我能读懂、能改、能解释”。

完成要求：

1. C 语言基础恢复到能看懂项目代码  
   必须掌握：变量、函数、数组、指针基础、结构体、头文件、宏、`typedef`、回调函数的基本概念。

2. ESP-IDF 能独立完成 5 个小实验  
   包括：点灯、按键、串口打印、定时器、一个传感器读取。

3. 能讲清老人看护器当前代码  
   每个文件干什么，每个主要函数干什么，正常/报警状态怎么切换。

4. AI 使用方式改变  
   不再直接问“帮我写完整代码”，而是问“解释这段”“考我理解”“帮我拆模块”“只写一个函数”。

过关标准：

> 你能不看 AI，用自己的话讲清楚当前项目主流程，并能独立改一个小功能，比如把报警条件、串口输出格式、按键逻辑改掉。

---

**阶段二：暑假，做出完整 IoT 闭环**

目标：把老人看护器从“串口显示”变成真正的联网智能硬件原型。

完成要求：

1. ESP32 能联网  
   会配置 WiFi，知道设备如何连接网络。

2. 能上传数据  
   至少掌握 HTTP 或 MQTT 其中一个。建议优先 MQTT，因为 IoT 常用。

3. 有真实传感器数据  
   比如人体感应、心率血氧、姿态检测、温湿度、烟雾、按键求助等，至少 2 个传感器。

4. 有报警状态机  
   不是简单 `if else` 糊起来，而是有明确状态：正常、预警、报警、已处理、故障。

5. 有手机/网页查看方式  
   不要求很高级，可以是简单网页后台、MQTT 面板、小程序原型、或者云平台仪表盘。

过关标准：

> 你能演示：老人看护器采集数据 → 判断异常 → ESP32 联网 → 上传状态 → 手机或网页看到结果 → 触发报警。

---

**阶段三：大三上，把项目变成简历作品**

目标：不是只做出来，而是做成别人一看就知道你有能力的项目。

完成要求：

1. 项目文档完整  
   包括：项目背景、系统架构图、硬件连接图、软件流程图、功能说明、调试记录。

2. 代码结构清晰  
   至少拆成：传感器模块、报警模块、联网模块、主逻辑模块。

3. 有演示视频  
   1 到 3 分钟，展示设备运行、异常触发、数据上传、手机/网页查看。

4. 有仓库  
   GitHub 或 Gitee 都可以。README 要写清楚项目怎么运行。

5. 能回答面试问题  
   比如：为什么用 ESP32？为什么用 MQTT/HTTP？报警误报怎么办？断网怎么办？AI 在项目中帮了什么？

过关标准：

> 你能把这个项目放进简历，并且面对老师、同学、面试官讲 5 分钟不虚。

---

**阶段四：大三下，冲实习**

目标：拿到 IoT / 嵌入式应用 / AI+硬件原型方向实习机会。

完成要求：

1. 简历有 1 个主项目 + 1 个小项目  
   主项目是老人看护器。小项目可以是 ESP32 智能灯、环境监测器、远程控制插座、MQTT 数据面板等。

2. 基础面试能过  
   必须掌握：C 指针基础、结构体、串口/I2C/SPI、WiFi、HTTP/MQTT、FreeRTOS 基础任务概念。

3. 能独立完成小需求  
   比如“加一个传感器”“把数据改成 JSON 上传”“断网自动重连”“报警阈值可配置”。

4. 开始投递  
   城市优先：南京、苏州、无锡、杭州、合肥。岗位关键词：`IoT应用开发`、`嵌入式应用开发`、`ESP32`、`物联网设备开发`、`智能硬件开发`。

过关标准：

> 你能投实习，不是靠“我很感兴趣”，而是靠项目、代码、演示、解释能力证明你能干活。
