# 全局设定 (Global Settings)

- **文件编码 (File Encoding)**: 始终使用 UTF-8 进行文件的读取、创建和写入。
- **注释语言 (Comment Language)**: 编写代码时，所有的注释和文档字符串必须使用中文。
- **时区 (Timezone)**: 在处理时间相关逻辑或分析上下文时，默认使用 东八区（南京时间，UTC+8）。
- **本地化 (Locale)**: 默认语言环境为 简体中文（zh-CN）。
- **强制问候 (Mandatory Greeting)**: 在每次回复用户的消息时，必须在开头第一句严格称呼：“Hello Wong.”

# Obsidian 课程资料整理规则 (Course Material Organization)

当你被要求向 Obsidian 中导入、整理或重排版课程资料时，必须自动遵循并应用以下标准化规则，无需用户反复叮嘱：

1. **强制数字前缀排序 (Mandatory Prefix Sorting)**:
   - **MOC 置顶**：每个学科根目录必须首先生生成一个名为 `00-[课程名]复习总览.md` 的核心 MOC 导航页。
   - **文件夹序列化**：为打破 Obsidian 默认的拼音/首字母打乱排序，所有的模块文件夹（如 `知识总结`、`习题集`）必须按照 MOC 中从上到下的认知逻辑顺序，强行赋予 `01-`、`02-` 等数字前缀（如 `01-知识总结`）。
   - **防重名前缀**：对于类似 `A卷答案.md` 等极易跨学科冲突的通用笔记，必须自动冠以学科前缀（如 `电工_21-22A卷答案.md`）。

2. **多形态文档精细化处理策略 (Smart Format Handling)**:
   - **文字型（DOCX / 纯文本）**：提取其核心文字内容，主动使用 AI 的理解能力重新编排 Markdown 层级结构，必须加入高亮（粗体）、列表和引用块（`> [!NOTE]` 等）提升美感。
   - **公式/图表/理科真题（PDF）**：**绝对禁止强行提取纯文本（会产生严重乱码）**。必须将 PDF 逐页切割为高清 PNG 存入所在文件夹的 `附件` 目录，并生成一份长卷 Markdown 用 `![[图片名.png]]` 语法串联，实现顺畅向下滚动的图文复习体验。
   - **超大型文献（复习全书 / 长篇PPT）**：无论资料多大，**全部将其拷贝收编进入对应科目的 `附件` 文件夹**，并在索引 Markdown 中使用纯正的内部双链 `[[文件名.pdf]]` 建立跳转。严禁使用外部绝对路径（file:///）链接，确保整个知识库 100% 独立便携。

3. **智能纠偏系统 (Smart Image Rotation)**:
   - 在导入手机拍摄的手写习题或卷子图片时，必须编写自动检测脚本读取宽高比。一旦发现“横躺”（宽>高）的照片，强制进行 90 度旋转矫正为竖版 (Portrait)，保证阅读不需要歪头。
   - 照片命名必须规范化、序列化（如 `[科目简称]习题_01.jpg`）。

4. **Frontmatter 元数据强制封印 (Metadata Enforcement)**:
   - 你创建或迁入的每一篇 Markdown 笔记，顶部必须打上标准的 YAML 属性，以确保未来的数据库检索和 Dataview 插件兼容：
     ```yaml
     ---
     type: course-note
     domain: course
     course: [学科全称]
     status: active
     tags: [[科目简称], 核心标签...]
     ---
     ```

# Markdown 格式与视觉排版偏好 (Markdown & Formatting)

1. **视觉排版优先**：重度使用 Obsidian Callout 提示块 (如 > [!tip], > [!warning]) 划分内容区块，并辅以 ==高亮文本== 和 **粗体** 来突出核心重点，保持版面极高可读性。
2. **万物皆可双链**：重度依赖 [[双向链接]] 连接碎片化知识点（Zettelkasten思想），让知识网自由生长，减少对死板层级文件夹的依赖。
3. **强制中文命名**：所有笔记文件的物理名称都必须是中文的，以保证阅读直观性（如有必须使用英文的专有名词，可中英结合）。
4. **代码导入专门规范**：导入的源码必须被统一包裹在一个标准的 Markdown 三反引号代码块中（例如 ` `python ... ` `），严禁出现格式断层，以防止代码内的 # 注释被错误渲染为标题。
